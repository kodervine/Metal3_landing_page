Manage your hardware more efficiently with Metal³

Metal³ enables users to see information about the hardware, set up and configure network connections, manage their infrastructure, among other tasks. It is similar to managing computer servers in a cloud environment like Google Cloud or Amazon Web Services. When a computer is manufactured, it typically only has basic hardware components such as the CPU, memory, and storage. Metal³ helps manage these bare metal systems and streamlines the process of managing hardware within a Kubernetes environment.

Metal³ Mission

What we do »

The Metal³ project is on a mission to provide a simple, reliable, and scalable way to manage bare metal infrastructure within a Kubernetes environment by simplifying the process of deployment and management to emulate virtualized infrastructure.



Goals of Metal³

1

Automating tasks that would otherwise be done manually.

2

Simplifying the process of deployment.

3

Improving the reliability of network configuration.

4

Hybrid experience for virtualized and bare metal infrastructure.

5

Reduce the complexity of managing infrastructure.

Metal³ components

Bare Metal Actuator

This helps manage the bare metal hosts within a Kubernetes cluster by listening for changes and translating those changes into actions.

Cluster API (CAPI) provider

This provides an easy and scalable way to deploy Kubernetes on bare metal hosts and enables administrators to manage their entire infrastructure using Kubernetes.

Bare Metal Operator

This tool helps keep track of the inventory of bare metal hosts, and also automatically discovers new hosts that need to be added to the system.

Ironic

This manages the lifecycle of physical servers in a data center. This includes automating tasks like server discovery and inventory management.

What makes Metal³ different?

Without Metal³ With Metal³
1 Hardware management done manually can be time-consuming. Bare Metal Host (BMH) operator automates hardware management tasks making it faster and less error-prone.
2 Significant manual configuration and setup when deployed with kubernetes. Cluster API (CAPI) provider for Metal³ automates the deployment and management of Kubernetes making it faster to set up and manage.
3 Network configuration done manually can cause errors and misconfigurations. Metal³ Network provisioning tools automate network configuration making it more reliable.
4 Node management can get complex and difficult to scale. Metal³ node management tools make it easier to manage and scale Kubernetes nodes.
